# DropSign v0.2 Design Spec

**Date:** 2026-05-08  
**Branch:** style/ui-polish  
**Author:** muminjun

---

## Overview

DropSign v0.2 adds flexible trigger modes, configurable signature style, mobile/touch polish, messages/localization, and lays a documented foundation for future trackpad freehand and pressure-sensitive drawing. All changes are additive and backward-compatible.

---

## Architecture

### New files

| File | Purpose |
|------|---------|
| `src/trigger.ts` | `resolveTrigger()`, `createFloatingTrigger()`, `createInlineTrigger()`, `attachCustomTrigger()` |
| `src/messages.ts` | `mergeMessages(defaults, user?)` — pure function, no DOM access |

### Modified files

| File | Changes |
|------|---------|
| `src/types.ts` | Add `DropSignTrigger`, `DropSignMessages`, `DropSignSignatureOptions` |
| `src/DropSign.ts` | Wire trigger/messages/signature options; pass merged config downstream |
| `src/signature-pad.ts` | Accept `messages` and `signatureOptions` args; use them in modal labels and `SignaturePad` constructor |
| `src/styles.ts` | Press effect CSS, viewport-fixed floating styles, inline trigger styles, mobile/touch polish |

### Unchanged files

`capture.ts`, `overlay.ts`, `placement.ts` — no changes unless a mobile CSS fix is needed in styles only.

## Types

```ts
export type DropSignTrigger =
  | {
      type?: 'floating';
      positionAnchor?: 'target' | 'viewport';   // default: 'target'
      position?: 'bottom-right' | 'bottom-left' | 'top-right' | 'top-left';
      label?: string;
      pressEffect?: boolean;                      // default: true
    }
  | {
      type: 'inline';
      container: string | HTMLElement;
      label?: string;
      variant?: 'button' | 'text';               // default: 'button'
      pressEffect?: boolean;
    }
  | {
      type: 'custom';
      element: string | HTMLElement;
    };

export interface DropSignMessages {
  sign?: string;
  clear?: string;
  cancel?: string;
  useSignature?: string;
  confirm?: string;
  delete?: string;
  signingTitle?: string;
  signingDescription?: string;
}

export interface DropSignSignatureOptions {
  penColor?: string;
  minWidth?: number;
  maxWidth?: number;
  velocityFilterWeight?: number;
}
```

`DropSignOptions` gains:
```ts
trigger?: DropSignTrigger;
messages?: Partial<DropSignMessages>;
signature?: DropSignSignatureOptions;
```

## Backward Compatibility

- `DropSign.init({ target, buttonText })` continues to work unchanged.
- No `trigger` → treated as `{ type: 'floating', positionAnchor: 'target' }`.
- Label priority: `trigger.label` > `messages.sign` > `buttonText` > `'Sign'`.
- `positionAnchor: 'target'` = current behavior (button inside `targetEl`, `position: absolute`).
- `positionAnchor: 'viewport'` = opt-in chat-widget style (appended to `document.body`, `position: fixed`).

---

## Trigger Modes

### Floating (`positionAnchor: 'target'`, default)
Button appended to `targetEl`, `position: absolute`, corner set by `position` option (default `bottom-right`).

### Floating (`positionAnchor: 'viewport'`)
Button appended to `document.body`, `position: fixed`, corner set by `position` option.  
`destroy()` removes button from `document.body`.

### Inline
Button created inside `container`. `variant: 'text'` renders as text-style link; `variant: 'button'` (default) renders as compact button.

### Custom
No new button created. Click listener attached to existing `element`. `destroy()` removes listener only — element remains in DOM.

---

## CSS Changes (styles.ts)

- Press effect: `.ds-button:active`, `.ds-btn-inline:active` → `transform: scale(0.96); transition: transform 0.1s ease`. Suppressed via `data-no-press` attribute when `pressEffect: false`.
- Viewport-fixed floating: `.ds-button-viewport` class, `position: fixed`, corner via inline style.
- Inline button: `.ds-btn-inline` class.
- Inline text variant: `.ds-btn-inline--text` — no background, underline style, no border.
- Mobile: `@media (max-width: 480px)` on `.ds-modal` for full-width layout. `@media (pointer: coarse)` on `.ds-resize-handle` for larger touch targets.

## Signature Options

Defaults: `penColor: '#111827'`, `minWidth: 0.7`, `maxWidth: 2.5`, `velocityFilterWeight: 0.7`.  
Passed directly to `new SignaturePad(canvas, { penColor, minWidth, maxWidth, velocityFilterWeight })`.

## Tests

- `buttonText` backward compat (existing, keep)
- Default floating: button inside `targetEl`
- `positionAnchor: 'viewport'`: button on `document.body`, not inside `targetEl`
- `position: 'bottom-left'`: correct CSS class or inline style
- Inline trigger: button created inside container
- Inline text variant: text-style class present
- Custom trigger: no new button; existing element survives destroy
- `destroy()` removes SDK triggers; custom element not removed
- Messages: trigger label and modal title reflect custom messages
- Signature option normalization: pure function test on mergeSignatureOptions (no DOM needed)

## Trackpad Freehand & Pressure-Sensitive Drawing

**v0.2 decision: not implemented.**

`signature_pad` v4 uses its own internal event handlers and does not expose a pressure or freehand hook without monkey-patching internals, creating a brittleness risk on library updates.

Both features are documented in README under Limitations and Roadmap.

## Data Attribute Foundation

Documented in README only for v0.2. Future concept:
```html
<div data-dropsign-target>
  <button data-dropsign-trigger>(서명)</button>
  <span data-dropsign-field="signature">(서명)</span>
</div>
```

`custom` trigger already supports CSS selector strings, so `[data-dropsign-trigger]` works today.
